# Love Court 微信小程序风险说明

> 版本：v1.0.0
> 适用平台：微信小程序
> 用途：提交微信审核时附带的已知风险与应对说明，便于审核团队评估。

## 一、风险概览

| 等级 | 风险项 | 影响 | 应对 |
| --- | --- | --- | --- |
| 高 | AI 生成内容合规风险 | 裁决可能包含不当内容 | 敏感词拦截 + AI prompt 限制 + 来源标识 |
| 高 | 用户输入敏感内容 | 危机/违法/医疗/投资内容 | 双层拦截（同步 + 宣判） |
| 中 | AI 服务不可用 | 无法生成 AI 裁决 | 自动回退本地规则 |
| 中 | 单实例文件存储 | 并发上限与水平扩展 | 限流 + 串行化写入 + 后续迁移 DB |
| 中 | 海报生成失败 | 分享体验受损 | 错误提示 + 缓存 + 重试 |
| 低 | 弱网/低端机体验 | 加载慢或操作失败 | 弱网提示 + 本地兜底 |
| 低 | 邀请码滥用 | 未授权加入 | 一次性 + 7 天过期 + 限流 |

## 二、详细风险与应对

### 2.1 AI 生成内容合规风险（高）

**风险描述**：
本小程序使用 DeepSeek AI 模型生成娱乐裁决。AI 生成内容可能存在偏差、不当表述或被诱导产生违规内容。

**应对措施**：

1. **AI Prompt 限制**：系统 prompt 明确禁止提供法律咨询、医疗建议、投资建议、危机干预、代码生成或泛闲聊，仅处理情侣/朋友/室友之间的轻量互动争议。
2. **敏感词前置拦截**：在用户输入阶段（`sanitizeCasePatch`）和宣判阶段（`validateCase`）双层拦截敏感词，覆盖：
   - 危机类：自杀、自残、伤害自己、轻生、想死、报警、家暴、殴打、虐待
   - 违法类：诈骗、赌博、毒品、大麻、枪支
   - 医疗类：诊断、用药、处方
   - 投资类：投资、股票、基金、理财、借贷、高利贷
   - 代码类：代码、编程、sql、password
   - 隐私类：身份证号、银行卡号
3. **来源标识**：裁决书正面明确标注「本裁决由 {model} AI模型生成」或「本裁决根据本地规则生成」，用户可识别内容来源。
4. **AI 内容说明**：隐私页提供 AI 生成内容说明，告知用户裁决仅为娱乐性质，可能存在偏差。
5. **回退机制**：AI 接口异常时自动回退本地规则，不向用户暴露原始 AI 错误。
6. **娱乐定位**：产品定位为娱乐向，裁决不构成严肃判断依据，已在用户协议中明确。

**残留风险**：AI 生成内容仍可能存在小概率不当表述，依赖敏感词库持续更新与用户举报机制。

### 2.2 用户输入敏感内容风险（高）

**风险描述**：
用户可能在陈词、昵称、案由中输入违法、暴力、色情、自伤自杀、人身攻击等内容。

**应对措施**：

1. **双层拦截**：
   - 同步阶段（`sanitizeCasePatch`）：写入时拦截，返回 400「内容包含敏感词，已被拦截」
   - 宣判阶段（`validateCase`）：宣判前再次拦截，避免绕过同步直接宣判
2. **关键词库**：覆盖危机、违法、医疗、投资、代码、隐私 6 大类敏感词。
3. **用户协议**：明确禁止上传违法/暴力/色情/自伤自杀/人身攻击内容，系统将拦截并拒绝。
4. **长度限制**：陈词 ≤ 500 字，案由/昵称/补充回答 ≤ 220 字，减少攻击面。
5. **白名单字段**：`sanitizeCasePatch` 仅接受 7 个白名单字段，忽略其他字段。

**残留风险**：关键词库无法覆盖所有变体（如拼音、谐音、拆字），需持续迭代。建议后续接入微信内容安全检测 API（`security.msgSecCheck`）。

### 2.3 AI 服务不可用风险（中）

**风险描述**：
DeepSeek API 可能因余额耗尽、接口故障、网络异常等原因不可用。

**应对措施**：

1. **自动回退**：`buildAiVerdict` 在 API 不可用时回退 `local-rules-fallback`，用户仍可获得裁决。
2. **provider 标识**：回退裁决标注 `local-rules-fallback`，与正常 AI 裁决区分。
3. **余额监控**：运维监控 DeepSeek 余额，低于阈值告警。
4. **不重复调用**：已有裁决的案件复用 `latestVerdictId`，不重复消耗 AI 调用。

**残留风险**：回退为本地规则，裁决质量与多样性下降，但不影响核心流程可用性。

### 2.4 单实例文件存储风险（中）

**风险描述**：
当前后端使用单实例文件型 JSON 存储（`data/cases.json`），存在并发上限、水平扩展困难、单点故障风险。

**应对措施**：

1. **串行化写入**：`withStoreWrite` 通过 Promise 链串行化所有写入，避免并发冲突。
2. **限流**：120 次/分钟限流，防止过载。
3. **自动备份**：每次写入节流备份，保留 14 份。
4. **灾备演练**：`scripts/drill.js` 验证备份与恢复链路。
5. **RPO/RTO**：RPO ≤ 1 小时（cron 备份），RTO ≤ 10 分钟（恢复 + 重启）。
6. **迁移预案**：数据模型已按 6 张表结构组织（users/cases/case_participants/case_statements/verdicts/case_access_tokens），便于后续迁移到真实 DB。

**残留风险**：单实例无法水平扩展，适合小规模灰度（< 1000 DAU）。规模化需迁移 DB。

### 2.5 海报生成失败风险（中）

**风险描述**：
海报生成依赖后端 sharp 库将 SVG 转 PNG，可能因内存不足、依赖缺失、SVG 渲染异常失败。

**应对措施**：

1. **错误提示**：失败时 toast「海报生成失败，请稍后重试」。
2. **缓存**：成功生成的海报缓存 30 天（`POSTER_CACHE_TTL_MS`），减少重复生成。
3. **本地兜底限制**：本地案件（`local-` 前缀）明确提示不支持海报。
4. **替代方案**：用户仍可「复制裁决文字」或「转发给对方」完成分享。

**残留风险**：海报生成失败时分享体验下降，但有替代方案。

### 2.6 弱网与低端机体验风险（低）

**风险描述**：
低端机或弱网条件下，页面加载慢、操作失败。

**应对措施**：

1. **弱网提示**：`subscribeNetworkStatus` 监听 2g/3g/unknown/none 网络，顶部显示提示。
2. **加载/错误态**：各页提供 loading/empty/error 状态卡片与重试按钮。
3. **本地兜底**：后端不可达时 fallback 到本地存储，不白屏。
4. **包体积控制**：生产构建 `minified: true`，主包 ≤ 2MB。
5. **性能目标**：首屏 ≤ 2s（WiFi）/ ≤ 4s（4G）。

**残留风险**：极端弱网下 AI 裁决可能超时（> 10s），依赖回退机制。

### 2.7 邀请码滥用风险（低）

**风险描述**：
邀请码可能被枚举或转发给非预期用户。

**应对措施**：

1. **一次性使用**：inviteToken 使用后标记 `usedAt`，不可重复使用。
2. **7 天过期**：`INVITE_TOKEN_TTL_MS = 7 天`，过期失效。
3. **限流**：120 次/分钟限流，防止暴力枚举。
4. **角色占用**：被告角色被占用时返回 409，不可重复绑定。
5. **权限隔离**：即使加入案件，也只能访问该案件，无法访问其他案件。

**残留风险**：6 位邀请码理论上可被暴力枚举，但限流 + 一次性使用显著降低风险。

### 2.8 数据删除合规风险（低）

**风险描述**：
用户删除数据后，备份中仍可能保留数据副本。

**应对措施**：

1. **软删除**：`deletedAt` 标记，列表不可见，可恢复（满足用户「撤回」需求）。
2. **彻底删除**：`purge` 物理删除案件及关联数据。
3. **删除我的数据**：隐私页提供全量软删除入口。
4. **备份保留**：备份保留 14 份（约 14 小时 ~ 14 天），过期自动裁剪。
5. **合规说明**：隐私政策说明备份数据的保留与销毁策略。

**残留风险**：备份中短期内仍存在已删除数据副本，符合「合理期限内删除」的合规要求。

## 三、风险监控与应急

### 3.1 监控指标

| 指标 | 告警阈值 | 应对 |
| --- | --- | --- |
| 接口错误率 | > 5% 持续 5 分钟 | 排查后端日志 |
| 429 限流次数 | > 1000/小时 | 评估是否调整限流参数 |
| AI 回退次数 | > 20% | 检查 DeepSeek 余额与接口 |
| 敏感词拦截次数 | > 100/小时 | 评估是否被刷量 |
| 磁盘使用 | > 90% | 清理备份 / 扩容 |
| 用户投诉 | ≥ 1 | 24h 内响应 |

### 3.2 应急预案

| 场景 | 应对 |
| --- | --- |
| AI 大量回退 | 检查 DeepSeek 余额，必要时临时关闭 AI 路径（清空 DEEPSEEK_API_KEY） |
| 后端 OOM | 重启进程，检查 `data/cases.json` 大小，必要时迁移 DB |
| 数据损坏 | `node scripts/restore.js --latest` 恢复 |
| 敏感内容泄露 | 紧急下架版本，更新敏感词库 |
| 微信审核驳回 | 按驳回理由修改后重新提交 |

## 四、风险承诺

1. 本小程序不提供法律、医疗、心理、投资咨询，仅为娱乐向产品。
2. AI 生成内容已做敏感词拦截与来源标识，但不保证 100% 准确性。
3. 用户数据按参与关系最小权限隔离，提供删除入口。
4. 服务端日志脱敏处理，不记录敏感个人信息明文。
5. 已建立备份与灾备机制，RPO ≤ 1 小时，RTO ≤ 10 分钟。
